# INTERNAL SECURITY ANALYSIS - CVE-2017-15361
# Classification: CONFIDENTIAL
# Analyst: Dr. Sarah Chen, Crypto Division
# Date: 2024-11-20

## Summary
The Infineon RSA Library v1.02.013 uses a "fast prime" generation
algorithm that produces primes with a predictable structure.

## Technical Details
Affected primes follow the pattern:
    p = k * M + (65537^a mod M)

Where:
    M = product of first N consecutive small primes (primorial)
    k, a = integers sampled during generation

This structure means the prime factors of the modulus N can be
recovered using a variant of Coppersmith's lattice-based attack,
FAR more efficiently than brute-force factoring.

## Detection
A key is ROCA-vulnerable if, for each small prime p_i,
the residue (N mod p_i) belongs to the multiplicative subgroup
generated by 65537 modulo p_i.

Known detection tools:
    - roca-detect (https://github.com/crocs-muni/roca)
    - RsaCtfTool --attack roca

Bit length  = 512 bits
Status      = *** VULNERABLE ***

## Factoring Difficulty
For 512-bit ROCA keys, factoring is feasible with:
    - Standard number field sieve (minutes)
    - Coppersmith's attack (seconds)
    - Online services like factordb.com

## Recommended Actions
1. Revoke ALL RSA keys generated by Infineon TPMs with firmware < 2.0
2. Re-generate keys using patched firmware or alternative HSM
3. Consider the intercepted communication COMPROMISED

## NOTE TO ANALYST
If you need to factor the key yourself, the math is:
    n = p * q
    phi(n) = (p-1) * (q-1)  
    d = e^(-1) mod phi(n)
    plaintext = ciphertext^d mod n
    flag = int_to_bytes(plaintext)
